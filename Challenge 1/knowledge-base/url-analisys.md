# 1. Domínio

Domínio muito recente → alto risco de phishing.

Domínio descartável ou temporário → suspeito.

Domínio hospedado em CDNs → pode ser usado por atacantes.

Domínos que imitam marcas famosas → paypal - paypa1.

# 2. Presença de IP

URLs que usam endereços IP em vez de nomes de domínio podem indicar phishing.

Menos comum, mas ainda relevante.

# 3. Parâmetros Encriptados

Parâmetros codificados ou criptografados no URL → possível tentativa de ocultar informações ou enganar filtros.

# 4. Substituição de Caracteres

Uso de caracteres similares ou alfabetos alternativos (ex.: cirílico) para imitar domínios legítimos.

Pode incluir trocas simples de letras (micros0ft.com), ou homógrafos complexos.

# 5. Comprimento excessivo

URLs muito longos e complicados podem indicar tentativa de esconder o verdadeiro domínio:

Ex.: https://secure-login.paypal.com.user-verification.account-update.com/login

# 6. TLD suspeito ou fora do padrão

Domínios legítimos de bancos ou grandes empresas geralmente usam .com, .com.br, .org.

Um TLD estranho como .xyz, .top, .club pode ser sinal de alerta.
